Create `backend/app/services/deep_research/prompts.py` with prompt templates from 05-RESEARCH.md:

1. `TLDR_PROMPT` -- Takes title, abstract, year, venue. Returns JSON with tldr_en, tldr_zh, methods list, datasets list, key_metrics dict, paper_type enum. Use XML-tag delimiters for paper content (prompt injection prevention per research pitfall #5).
2. `DEEP_ANALYSIS_PROMPT` -- Takes title, abstract, full parsed_content sections. Returns JSON with detailed_methodology (str), key_contributions (list[str]), limitations (list[str]), future_work (list[str]). For top-N papers with parsed full text.
3. `RELATIONSHIP_PROMPT` -- Takes two papers (title + abstract each). Returns JSON with relationship type (improvement/comparison/survey/application/theoretical_basis/unrelated), confidence (0-1), explanation (str). From research doc.

All prompts end with "Return ONLY valid JSON, no additional text." Use double-brace escaping for JSON template literals in f-strings. Truncate abstracts to 1000 chars in prompt builder functions.

Create `backend/app/services/deep_research/analyzer.py`:

1. `PaperAnalysis` Pydantic model: paper_id (str), tldr_en (str), tldr_zh (str), methods (list[str]), datasets (list[str]), key_metrics (dict), paper_type (str), detailed_methodology (str | None), key_contributions (list[str] | None), limitations (list[str] | None).

2. `async def screen_paper(paper: Paper, session: AsyncSession, user_id: str) -> PaperAnalysis`:
   - Build messages from TLDR_PROMPT with paper.title, paper.abstract[:1000], paper.year, paper.venue
   - Call `llm_completion` with model="claude-haiku-4-20250514", max_tokens=512, request_type="deep_research_screening"
   - Parse JSON response into PaperAnalysis. On JSON parse error, return PaperAnalysis with empty fields and paper_type="unknown".

3. `async def deep_analyze_paper(paper: Paper, session: AsyncSession, user_id: str) -> PaperAnalysis`:
   - Build messages from DEEP_ANALYSIS_PROMPT with paper.title, paper.abstract, paper.parsed_content
   - Call `llm_completion` with model=None (default Sonnet), max_tokens=2048, request_type="deep_research_analysis"
   - Parse and merge results into existing PaperAnalysis (from screening pass).

4. `async def analyze_papers_tiered(papers: list[Paper], session: AsyncSession, user_id: str, top_n: int = 20, cost_ceiling: float = 10.0) -> list[PaperAnalysis]`:
   - Pass 1: Screen ALL papers with `screen_paper` (Haiku). Track cumulative cost. Abort if cost_ceiling exceeded.
   - Pass 2: Sort papers by quality score (from DB or default 0). Take top_n with parsed_content not None. Call `deep_analyze_paper` for each. Merge into screening results.
   - Return list of PaperAnalysis.

5. `RelationshipResult` Pydantic model: paper_a_id (str), paper_b_id (str), relationship (str), confidence (float), explanation (str).

6. `async def classify_relationships(paper_pairs: list[tuple[Paper, Paper]], session: AsyncSession, user_id: str) -> list[RelationshipResult]`:
   - For each pair, build RELATIONSHIP_PROMPT, call llm_completion with Haiku, parse JSON.
   - Only classify pairs where both papers have abstracts. Skip if either abstract is None.
   - Return list of RelationshipResult.

Use `json.loads` for parsing LLM responses. Wrap in try/except JSONDecodeError with fallback defaults. Log warnings on parse failures. All functions are pure async -- no mutation of input objects.

Reference: AI-Scientist generate_ideas.py for structured LLM prompt patterns, gpt-researcher for tiered analysis.
