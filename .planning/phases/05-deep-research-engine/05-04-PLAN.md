**Add** to `backend/app/schemas/deep_research.py`:

1. `RefineRequest(BaseModel)`: task_id (str), exclude_methods (list[str] | None), exclude_paper_types (list[str] | None), year_from (int | None), year_to (int | None), reanalyze (bool = True). Represents filter adjustments that trigger partial re-analysis.

2. `ExpandAreaRequest(BaseModel)`: task_id (str), paper_ids (list[str], min 1, max 10), depth (int = 1, ge=1, le=2), budget (int = 50, ge=10, le=200). Represents manual expansion of specific graph nodes.

3. `RefineResponse(BaseModel)`: task_id (str), workflow_id (str | None), filtered_count (int), message (str).

4. `ExpandAreaResponse(BaseModel)`: task_id (str), new_papers (int), new_edges (int), message (str).

**Add** to `backend/app/routers/deep_research.py`:

1. `POST /deep-research/tasks/{task_id}/refine`:
   - Requires auth. Verify task ownership.
   - Load DeepResearchTask. Verify status == "completed".
   - Load paper analyses from task config["paper_analyses"].
   - Apply filters: exclude papers whose methods intersect exclude_methods, exclude by paper_type, filter by year range.
   - Update task config with new filter settings (stored for reproducibility).
   - If reanalyze=True: start a new Temporal workflow for re-analysis (only analyze + classify + gaps + report activities on filtered subset). Use workflow_id = f"deep-research-refine-{task_id}-{uuid4().hex[:8]}". Update task status to "running".
   - If reanalyze=False: just return filtered count without re-analysis.
   - Return RefineResponse.

2. `POST /deep-research/tasks/{task_id}/expand`:
   - Requires auth. Verify task ownership.
   - Load DeepResearchTask. Verify status in ("completed", "running").
   - Create SemanticScholarClient and Neo4jClient (same pattern as citations router).
   - Call `expand_citations(paper_ids, s2_client, neo4j_client, depth=request.depth, budget_per_level=request.budget//request.depth, total_budget=request.budget)`.
   - Return ExpandAreaResponse with new_papers and new_edges counts from CitationGraph result.

3. `GET /deep-research/tasks/{task_id}/report`:
   - Requires auth. Verify task ownership.
   - Load DeepResearchTask. Return report_markdown as plain text (content-type text/markdown) or as JSON field.
   - Return 404 if report_markdown is None (task not yet completed).

Handle errors: 404 if task not found, 403 if not owner, 409 if task status doesn't allow operation (e.g., refine on a running task).
