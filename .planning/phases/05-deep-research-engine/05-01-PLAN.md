**Replace** `backend/app/workflows/deep_research.py` placeholder with the real workflow:

2. Define `DeepResearchWorkflowResult` dataclass with task_id, status, papers_found, papers_analyzed, total_cost.
3. `@workflow.defn class DeepResearchWorkflow` with `_progress` dict attribute (not Pydantic -- Temporal queries must return simple types).
4. `@workflow.query def get_progress(self) -> dict` returns self._progress.
5. `@workflow.run async def run(self, input: DeepResearchWorkflowInput)`:
   - Set progress phase="searching", call `search_papers_activity` with JSON-serialized search params
   - Set progress phase="expanding", call `expand_citations_activity` with paper IDs JSON
   - Set progress phase="scoring", call `score_papers_activity` with paper IDs JSON
   - Set progress phase="analyzing" (placeholder -- real analysis in Plan 02)
   - Set progress phase="completed"
   - Each activity: start_to_close_timeout=timedelta(minutes=5), retry_policy=RetryPolicy(maximum_attempts=3)
   - Update papers_found, papers_analyzed counts in progress after each activity

CRITICAL: All I/O in activities, NOT in workflow. Activities use JSON string I/O (Temporal payload constraint). Workflow only orchestrates.

**Update** `backend/app/workflows/activities.py` -- keep placeholder_search, ADD 3 new activities:

1. `search_papers_activity(input_json: str) -> str`: Parse JSON to get query/sources/limits. Create httpx.AsyncClient. Call `search_all_sources`. Persist papers to PostgreSQL (create async session from engine). Return JSON with paper IDs list and count.
2. `expand_citations_activity(input_json: str) -> str`: Parse JSON to get paper_ids, depth, budget. Create SemanticScholarClient + Neo4jClient. Call `expand_citations`. Return JSON with node count and edge count.
3. `score_papers_activity(input_json: str) -> str`: Parse JSON to get paper_ids. Create Neo4jClient. Call `score_papers_batch`. Persist quality scores. Return JSON with scored count.

Each activity creates its own clients/sessions (isolation pattern from Phase 3.1). Import from existing service modules.
