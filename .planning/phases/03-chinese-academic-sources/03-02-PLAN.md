1. Create `backend/app/services/paper_search/wanfang_client.py`:
   - Define `WanfangBlockedError(Exception)` for when Wanfang blocks the scraper.
   - `WanfangClient` inherits from `BasePaperClient`.
   - Constructor takes `http_client: httpx.AsyncClient` and `browser_pool: BrowserPool`.
   - **CSS selectors as module-level constants** using RESEARCH.md `WANFANG_SELECTORS` pattern. Include fallback selectors.
   - `search_keywords(query, limit=25)`:
     - URL: `https://s.wanfangdata.com.cn/paper?q={query}&style=detail`
     - Use `self._pool.get_context("wanfangdata.com.cn")` for browser context.
     - Navigate with `wait_until="networkidle"`, timeout 30000ms.
     - Get `page.content()`, parse with `_parse_results(html, limit)`.
     - If page loaded but 0 results parsed and page content is non-trivial, log `SELECTOR_MISMATCH` warning.
     - Always close page in `finally` block.
     - Rate limit: `await asyncio.sleep(3.0)` after each request.
   - `search_doi(doi)`: Return None (Wanfang does not support public DOI lookup).
   - `search_author(author, limit=25)`: Same URL pattern, Wanfang uses same search box.
   - `_parse_results(html, limit)`: Parse with BeautifulSoup + lxml. Extract from each `.normal-list-item`: title, authors, venue, year, abstract. Set `language="zh"`, `sources=[PaperSource.WANFANG]`. Extract wanfang_id from title link href if possible. Cap at `limit`.
   - Helper methods: `_safe_text`, `_extract_year`.

   **Key difference from CNKI:** Wanfang is less aggressive with anti-scraping (no CAPTCHA puzzle), but may block after sustained scraping. Wanfang shows abstracts on the search results page (CNKI may not). Use 3s delays instead of 4s.
