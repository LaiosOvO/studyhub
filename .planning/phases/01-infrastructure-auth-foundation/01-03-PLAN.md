Create `src/lib/api.ts` — API client:
- Base URL from NEXT_PUBLIC_API_URL env var (default: http://localhost:8000)
- `apiFetch(path, options)` wrapper that:
  - Adds Authorization header if access token exists
  - Auto-refreshes token on 401 response (single retry)
  - Returns parsed JSON
  - Handles race conditions: if multiple requests get 401, only one refresh happens (use a promise queue)

Create `src/lib/auth.ts` — Token management:
- Store access token in memory (module-level variable, NOT localStorage — XSS protection)
- Store refresh token in localStorage (acceptable for MVP, httpOnly cookie requires backend changes)
- `getAccessToken()` — returns current access token
- `setTokens(access, refresh)` — store both
- `clearTokens()` — remove both
- `refreshToken()` — call POST /auth/refresh, update stored tokens
- `isAuthenticated()` — check if access token exists and not expired (decode JWT client-side for expiry check)

Create `src/stores/auth-store.ts` — Zustand store:
```typescript
interface AuthState {
  user: User | null;
  isLoading: boolean;
  isAuthenticated: boolean;
  login: (email: string, password: string) => Promise<void>;
  register: (email: string, password: string, fullName: string) => Promise<void>;
  logout: () => Promise<void>;
  loadUser: () => Promise<void>;  // Called on app init to restore session
}
```
- `login` — call /auth/login, store tokens, fetch /auth/me, set user
- `register` — call /auth/register, then auto-login
- `logout` — call /auth/logout, clear tokens, set user to null
- `loadUser` — check for refresh token in localStorage, if exists try to refresh + fetch /auth/me (session persistence across refresh)

Create `src/app/[locale]/(auth)/layout.tsx`:
- Centered card layout for auth pages
