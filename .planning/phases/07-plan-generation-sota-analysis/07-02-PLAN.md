1. Create `backend/app/services/plan_generation/plan_generator.py`:
   - `async def generate_experiment_plans(context: PlanGenerationContext, session: AsyncSession, user_id: str, num_plans: int = 3, num_reflections: int = 3) -> list[dict]`:
     AI-Scientist-style generation with reflection. For each plan (up to num_plans):
     a. Build initial prompt with build_plan_generation_prompt, passing SOTA summary, gaps, improvements, and entry-specific context.
     b. Call llm_completion (Sonnet, request_type="plan_generation", max_tokens=4096).
     c. Parse JSON response into plan draft dict.
     d. Reflection loop (2 to num_reflections): call build_reflection_prompt with current draft, llm_completion (Sonnet, request_type="plan_reflection"). If response contains "I am done", break. Otherwise parse as updated draft.
     e. Append final draft to results list.
     Each plan generation is independent (could parallelize in future, but sequential for cost control in v1).
   - Helper `def _format_sota_summary(sota: SOTAResult | None) -> str`: Formats SOTAResult into readable text for prompts.
   - Helper `def _format_gaps_summary(gaps: list[dict]) -> str`: Formats gap list into readable text.
   - Helper `def _format_improvements_summary(improvements: list[ImprovementOpportunity]) -> str`: Formats improvements.
   - Helper `def _format_entry_context(context: PlanGenerationContext) -> str`: Returns extra context based on focus_paper or focus_gap if present.
   - All functions are pure async or pure functions. No mutation of inputs.
   - Error handling: if any single plan fails to parse, log warning and continue to next. Return whatever plans succeeded.

2. Create `backend/app/services/plan_generation/feasibility_scorer.py`:
   - `async def score_feasibility(plan_draft: dict, direction: str, session: AsyncSession, user_id: str) -> FeasibilityScore`:
     Uses build_feasibility_prompt + llm_completion (Haiku for cost efficiency, request_type="plan_feasibility", max_tokens=1024).
     Parses JSON into FeasibilityScore. Computes `overall` as weighted average: compute_requirements*0.25 + data_availability*0.25 + expected_improvement*0.25 + (6-difficulty)*0.25 (inverted so higher = better).
     Returns default FeasibilityScore(compute_requirements=3, data_availability=3, expected_improvement=3, difficulty=3, overall=3.0, explanation="Scoring unavailable") on failure.
   - `async def score_plans_batch(plan_drafts: list[dict], direction: str, session: AsyncSession, user_id: str) -> list[FeasibilityScore]`:
     Scores all plans sequentially (Haiku is cheap). Returns list aligned with input.

3. Create `backend/app/services/plan_generation/dataset_recommender.py`:
   - `async def recommend_datasets(direction: str, plan_datasets: list[str], data_strategy: str, limit: int = 10) -> list[DatasetRecommendation]`:
     Uses `huggingface_hub.HfApi().list_datasets(search=keyword, sort="downloads", direction=-1, limit=limit)` wrapped in `asyncio.to_thread` (HfApi is sync, same pattern as pyalex/scholarly).
     Extract search keywords from direction (split, take nouns/key terms, limit to 3 keywords).
     Search HF Hub for each keyword, collect results.
     Deduplicate by dataset ID, rank by downloads.
     If data_strategy == "own_data", return empty list (user provides their own).
     If data_strategy == "hybrid", include both HF results and note that user will add their own.
     Non-fatal: return empty list on any HfApi error, log warning.
   - `def extract_search_keywords(direction: str) -> list[str]`: Split direction into meaningful search terms. Remove common stop words. Return top 3 terms.
   - `def deduplicate_and_rank(results: list[DatasetRecommendation], limit: int) -> list[DatasetRecommendation]`: Deduplicate by name, sort by downloads descending, return top `limit`.

Reference: AI-Scientist `generate_ideas.py` for reflection pattern, MLE-agent `planner.py` for plan structure.
