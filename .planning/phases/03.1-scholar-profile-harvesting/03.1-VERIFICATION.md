# Phase 03.1 Verification: Scholar Profile Harvesting

## Goal Check

**Goal**: Build a scholar profile database seeded from Baidu Baike and Google Scholar, storing researcher metadata (name, institution, rank, age, research fields, h-index) linked to papers.

**Result**: PASSED

## Must-Have Verification

### 1. Scholar model stores name, institution, title/rank, birth_year, research_fields, h_index, and source URLs
- **Status**: PASSED
- **Evidence**: `backend/app/models/scholar.py` defines Scholar with all required fields
- **Verified**: `from app.models.scholar import Scholar` succeeds; model has name, institution, title (JSON), rank, birth_year, research_fields (JSON), h_index, source_urls (JSON), plus additional fields (honors, education, linked_paper_ids, google_scholar_id)

### 2. Baidu Baike scraper extracts scholar profiles with normalized metadata
- **Status**: PASSED
- **Evidence**: `backend/app/services/scholar/baike_scraper.py` implements BaikeScraper with CSS selector fallbacks
- **Verified**: `from app.services.scholar.baike_scraper import BaikeScraper` succeeds; scrape_profile method accepts name + optional URL, returns ScholarCreate or None

### 3. Google Scholar enrichment adds h-index, citation count, and publication list
- **Status**: PASSED
- **Evidence**: `backend/app/services/scholar/google_scholar_enricher.py` implements GoogleScholarEnricher
- **Verified**: `from app.services.scholar.google_scholar_enricher import GoogleScholarEnricher` succeeds; enrich_scholar updates h_index, total_citations, google_scholar_id; rate-limited with semaphore + 5s cooldown

### 4. Scholars are linked to papers via author name matching with fuzzy CJK support
- **Status**: PASSED
- **Evidence**: `backend/app/services/scholar/paper_linker.py` implements fuzzy_name_match and link_scholar_papers
- **Verified**: `fuzzy_name_match('葛均波', '葛均波')` returns True; `fuzzy_name_match('Zhang San', 'zhang san')` returns True; CJK uses exact match, English uses rapidfuzz ratio >= 80

### 5. Seed data for ECG domain scholars is imported and verified
- **Status**: PASSED
- **Evidence**: `backend/app/services/scholar/seed_importer.py` reads `data/seed/ecg_scholars.json` (10 scholars) and upserts via ON CONFLICT
- **Verified**: `from app.services.scholar.seed_importer import import_seed_scholars` succeeds; POST /scholars/import-seed endpoint registered
