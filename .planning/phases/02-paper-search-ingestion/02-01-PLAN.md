  <action>
    **Before coding:** Run `gitnexus query gpt-researcher "paper search retriever"` and `gitnexus query MLE-agent "arxiv search"` to study reference patterns.

    **Install dependencies:** Add pyalex, rapidfuzz, tenacity to backend/pyproject.toml. Run `cd backend && uv add pyalex rapidfuzz tenacity`.

    **Add config settings** to backend/app/config.py:
    - `openalex_api_key: str = ""` (required, free from openalex.org)
    - `s2_api_key: str = ""` (recommended for dedicated rate limit)
    - `pubmed_api_key: str = ""` (recommended for 10 RPS)
    - `grobid_url: str = "http://localhost:8070"` (for Plan 03)

    **Create backend/app/schemas/paper.py:**
    - `PaperSource` enum: OPENALEX, SEMANTIC_SCHOLAR, PUBMED, ARXIV
    - `PaperResult` Pydantic model (unified schema from RESEARCH.md Pattern 1): doi, openalex_id, s2_id, pmid, arxiv_id, title, abstract, authors (list[str]), year, venue, language, citation_count, pdf_url, is_open_access, sources (list[PaperSource])
    - `PaperCreate` for DB persistence (extends PaperResult with optional parsed_content, pdf_storage_key)

    **Create backend/app/schemas/search.py:**
    - `SearchType` enum: KEYWORD, DOI, TITLE, AUTHOR
    - `SearchRequest`: query (str), search_type (SearchType, default KEYWORD), limit (int, default 25, max 100), sources (optional list[PaperSource] to filter which sources)
    - `SearchResponse`: papers (list[PaperResult]), total (int), sources_queried (list[PaperSource]), sources_failed (list[PaperSource])

    **Create backend/app/services/paper_search/base_client.py:**
    - Abstract `BasePaperClient` with httpx.AsyncClient injected (RESEARCH.md Pattern 4)
    - Abstract methods: search_keywords, search_doi, search_author -- all return list[PaperResult] or PaperResult|None
    - Each method wrapped with tenacity retry (3 attempts, exponential backoff, retry on httpx.HTTPStatusError with 429/500/502/503)

    **Create 4 source clients** (each inherits BasePaperClient):

    1. `openalex_client.py`: Use pyalex (sync) via `asyncio.to_thread`. Configure pyalex.config.api_key from settings. Map OpenAlex response fields to PaperResult. Handle `abstract_inverted_index` -> plain text conversion. Support keyword, DOI (filter by doi), and author (filter by authorships.author.display_name) searches.

    2. `s2_client.py`: Use httpx async directly to api.semanticscholar.org/graph/v1. Add x-api-key header if configured. Map S2 fields (paperId, externalIds, title, abstract, authors, year, citationCount, venue, isOpenAccess, openAccessPdf) to PaperResult. Support keyword (paper/search), DOI (paper/{DOI}), author (author/search then author/{id}/papers).

    3. `pubmed_client.py`: Use httpx async to eutils.ncbi.nlm.nih.gov. Two-step: esearch (get PMIDs) then efetch (get details as XML). Parse XML with xml.etree.ElementTree. Extract DOI from ArticleIdList. Map MedlineCitation fields to PaperResult. Support keyword (term parameter), DOI (term=DOI[AID]), author (term=author[au]).

    4. `arxiv_client.py`: Use httpx async to export.arxiv.org/api/query. Enforce 1 req/3 sec rate limit via asyncio.Semaphore(1) + asyncio.sleep(3). Parse Atom XML response. Map entry fields to PaperResult. Support keyword (search_query=all:query), DOI (not available -- return empty), author (search_query=au:author).

    **Create backend/app/services/paper_search/deduplicator.py:**
    - Three-tier dedup from RESEARCH.md Pattern 3
    - Tier 1: Exact DOI match (normalized lowercase, stripped)
    - Tier 2: Normalized title + year exact match (lowercase, strip whitespace/punctuation)
    - Tier 3: Fuzzy title match using `rapidfuzz.fuzz.ratio > 90`
    - On match: merge sources lists (create new PaperResult with combined sources, prefer non-None fields from the richer record)
    - Handle Chinese titles: detect CJK characters, use token-level comparison for CJK (split on characters rather than words)

    **Create backend/app/models/paper.py:**
    - SQLAlchemy Paper model from RESEARCH.md (all fields from PaperResult + parsed_content JSONB, pdf_storage_key, created_at, updated_at)
    - UUID primary key, unique index on doi, indexes on openalex_id, s2_id, pmid, arxiv_id

    **Update backend/app/models/__init__.py** to export Paper alongside existing User and LLMUsage.

    **Create Alembic migration** 003_create_papers_table.py for the papers table.

    - Fan out to all 4 clients using asyncio.gather with return_exceptions=True (RESEARCH.md Pattern 2)
    - For each source: if result is Exception, log warning and add to sources_failed list; otherwise extend all_papers
    - Call deduplicate() on combined results
    - Return SearchResponse with papers, total count, sources_queried, sources_failed
    - Support filtering by specific sources (if request.sources is set, only query those)
    - Use asyncio.Semaphore per source for rate limiting (OpenAlex: 10 concurrent, S2: 1 concurrent, PubMed: 3 concurrent, arXiv: 1 concurrent)

    **Create backend/app/routers/search.py:**
    - GET /search/papers with query params: q (str, required), type (SearchType, default keyword), limit (int, default 25), sources (optional comma-separated PaperSource values)
    - Inject shared httpx.AsyncClient via FastAPI dependency (create once at app startup in lifespan, store in app.state)
    - Call search_all_sources with constructed SearchRequest
    - Return ApiResponse[SearchResponse]
    - Add rate limit: 30 req/min per IP (generous since this fans out to 4 sources)
