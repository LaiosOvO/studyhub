   ```python
   class ReadingList(Base):
       __tablename__ = "reading_lists"
       id: Mapped[str] = mapped_column(String(36), primary_key=True, default=lambda: uuid.uuid4().hex)
       user_id: Mapped[str] = mapped_column(String(36), nullable=False, index=True)
       name: Mapped[str] = mapped_column(String(200), nullable=False)
       description: Mapped[str | None] = mapped_column(Text, nullable=True)
       paper_ids: Mapped[list] = mapped_column(JSON, default=list)  # List of S2 paper IDs
       created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
       updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())
   ```

2. Create `backend/app/schemas/reading_list.py`:
   - ReadingListCreate: name (str, 1-200 chars), description (optional str), paper_ids (optional list[str])
   - ReadingListUpdate: name (optional), description (optional), paper_ids (optional)
   - ReadingListResponse: id, user_id, name, description, paper_ids, created_at, updated_at (with from_attributes)
   - AddPaperRequest: paper_id (str)
   - RemovePaperRequest: paper_id (str)

3. Create `backend/app/routers/reading_lists.py`:
   - POST /reading-lists -- create new list (owner = current user)
   - GET /reading-lists -- list user's reading lists (paginated: skip, limit params)
   - GET /reading-lists/{list_id} -- get single list (ownership check: 404 if not owner)
   - PUT /reading-lists/{list_id} -- update list (ownership check)
   - DELETE /reading-lists/{list_id} -- delete list (ownership check)
   - POST /reading-lists/{list_id}/papers -- add paper_id to list (append to paper_ids, skip if already present, immutable: create new list)
   - DELETE /reading-lists/{list_id}/papers/{paper_id} -- remove paper_id from list (immutable: create new list without that ID)
   All endpoints require authentication via get_current_user. All return ApiResponse envelope. All use immutable patterns (create new list objects, never mutate in place).

1. Create `FilterBar.tsx`:
   - Horizontal bar above the visualization (flex row, gap-4, p-3, bg-white border-b).
   - Year range: two number inputs (from/to) with min 1900, max current year. On change, calls setFilters({ yearRange: [from, to] }). Clear button sets yearRange to null.
   - Quality threshold: range slider (0-1, step 0.1) showing current value. On change, calls setFilters({ qualityThreshold: value }).
   - Method type: multi-select dropdown (checkboxes). Options derived from unique methods in the current nodes data (passed as prop). On change, calls setFilters({ methodTypes: selectedMethods }).
   - All controls read current values from usePaperMapStore.filters.
   - Bilingual labels via useTranslations('paperMap').

2. Create `ExportMenu.tsx`:
   - Dropdown button (icon + "Export" text) that opens a menu with three options:
   - **JSON**: Serializes current filtered nodes+edges as JSON, creates Blob, triggers download as `research-graph-{date}.json`.
   - **CSV**: Converts nodes to CSV with columns: paper_id, title, year, citation_count, quality_score, cluster. Uses comma separator, handles commas in titles with quoting. Downloads as `research-papers-{date}.csv`.
   - **PNG**: Uses html-to-image `toPng` on the graph container element (passed as ref prop). Downloads as `research-graph-{date}.png`.
   - All export functions are pure (read data, produce output, no side effects on state).

3. Create `hooks/useReadingLists.ts`:
   - Fetches user's reading lists from `/api/reading-lists` on mount.
   - Returns `{ lists, isLoading, createList, addPaper, removePaper, deleteList, refresh }`.
   - createList(name, description?) -- POST /api/reading-lists.
   - addPaper(listId, paperId) -- POST /api/reading-lists/{listId}/papers.
   - removePaper(listId, paperId) -- DELETE /api/reading-lists/{listId}/papers/{paperId}.
   - deleteList(listId) -- DELETE /api/reading-lists/{listId}.
   - All mutations call refresh() after success to re-fetch list.
