# StudyHub — AI 驱动学术研究平台

## 1. 项目描述

StudyHub 是一个面向科研人员的 AI 驱动学术研究平台。用户输入研究方向或论文，平台自动检索全球文献、构建论文关系地图、AI 发现研究空白、生成并执行实验方案、匹配跨学科合作者。核心卖点是**论文地图可视化**和**全自动实验流水线**。

**目标用户**：高校研究生、科研人员、企业 R&D 团队  
**后端 API**：`http://101.126.141.165:8000`（JWT 认证）

## 2. 页面结构

- `/` — 首页（公开）
- `/search` — 论文搜索页（公开）
- `/papers/:paperId` — 论文详情页（公开）
- `/login` — 登录（未登录可访问）
- `/register` — 注册（未登录可访问）
- `/research/:taskId` — 论文地图页（需登录）
- `/experiments` — 实验仪表盘（需登录）
- `/community` — 社区（需登录）
- `/reading-lists` — 阅读列表（需登录）

## 3. 核心功能

- [x] 首页：三种研究入口（方向选择 / 论文输入 / AI 对话）
- [x] 论文搜索：多类型搜索、筛选、排序
- [x] 论文详情：全文信息、引用分析
- [ ] 论文地图：React Flow 引用图谱 / Deck.gl 主题地图 / 时间线
- [ ] 深度研究：AI 任务创建与实时进度 WebSocket
- [ ] 实验仪表盘：方案生成与执行
- [ ] 用户认证：注册/登录/JWT 管理
- [ ] 社区：学者匹配、协作者推荐

## 4. 后端集成计划

- **认证**：JWT Bearer Token（localStorage 存储 access_token + refresh_token）
- **Token 刷新**：401 自动调用 `/auth/refresh`，失败跳转 `/login`
- **WebSocket**：深度研究任务进度 + 站内消息实时推送
- **Supabase**：不需要（使用自有后端）

## 5. 开发阶段计划

### Phase 1：核心页面 UI（当前阶段）
- 目标：首页、搜索页、论文详情页的完整 UI

### Phase 2：用户认证
- 目标：登录/注册页面 + JWT 管理 + 路由守卫
- 产出：完整的认证流程

### Phase 3：论文搜索接入后端
- 目标：接入 `/search/papers` API，真实搜索结果

### Phase 4：论文地图可视化
- 目标：React Flow 引用图谱，Deck.gl 主题散点图
- 产出：核心可视化功能

### Phase 5：深度研究与实验流水线
- 目标：任务创建、WebSocket 进度、实验方案生成
- 产出：端到端研究流程
