# StudyHub 前端界面需求

## 项目定位

AI 驱动的学术研究平台：输入研究方向 → 自动检索全球文献、构建论文关系地图 → AI 发现研究空白 → 生成并自动执行实验方案 → 匹配跨学科合作者。

核心卖点是**论文地图可视化**和**全自动实验流水线**。

## 视觉参考

- [O-DataMap](https://o-datamap.oall.com/) — 交互式科学数据地图，可视化论文、学者和跨学科领域，参考其整体美感和交互风格
- 科研工具的克制美学：大量留白、信息密度高但不杂乱、数据可视化优先

## 后端 API

- **地址**: `http://101.126.141.165:8000`
- **文档**: `http://101.126.141.165:8000/docs` (Swagger)
- **认证**: JWT Bearer Token (access_token + refresh_token)
- **Token 存储**: localStorage
- **401 处理**: 自动用 refresh_token 换新 access_token，失败则跳转登录

### API 端点一览

## 路由结构

```
/[locale]/                              # 首页
/[locale]/search                        # 论文搜索（公开）
/[locale]/(auth)/login                  # 登录
/[locale]/(auth)/register               # 注册
/[locale]/(protected)/research/[taskId] # 论文地图（需登录）
/[locale]/(protected)/experiments       # 实验仪表盘（需登录）
/[locale]/(protected)/community         # 社区（需登录）
/[locale]/plans                         # 实验方案
/[locale]/papers/[paperId]              # 论文详情（公开）
/[locale]/(protected)/reading-lists     # 阅读列表（需登录）
/[locale]/(protected)/scholars          # 学者管理（需登录）
/[locale]/(protected)/research/[taskId]/report  # 文献综述报告（需登录）
```

- `(auth)` 路由组：未登录可访问，已登录自动跳转首页
- `(protected)` 路由组：未登录自动跳转 `/login`
- `[locale]` 支持 `zh-CN`（默认）和 `en`

## 页面设计

### 1. 首页 `/`

**目标**: 让用户 3 秒内理解平台价值，10 秒内开始使用。

**布局**:
- 顶部导航栏：Logo、搜索、论文地图、实验、社区、语言切换、登录/用户头像
- Hero 区域：
  - 大标题："从论文到实验，AI 全程加速你的研究"
  - 副标题："输入研究方向，自动检索文献、发现空白、生成并执行实验方案"
  - **三种研究入口**（Tab 或卡片切换）：
    1. **选择研究方向**：下拉/搜索选择预置的专业方向（如"自然语言处理"、"计算机视觉"、"强化学习"等），每个方向关联种子论文和种子关键词，选择后自动启动多轮搜索
    2. **输入论文**：粘贴论文标题/DOI/URL，系统以此论文为种子，自动扩展搜索相关论文
    3. **AI 对话**：聊天式界面，用户用自然语言描述研究兴趣（如"我想研究大模型在医疗影像中的应用"），AI agent 理解意图后自动拆分关键词、确定搜索策略、多轮搜索论文
  - 每种入口最终都触发"开始研究"→ 进入深度研究流程
- 功能卡片区（3 列）：
  - 论文地图：可视化数千篇论文的引用关系和研究脉络
  - AI 实验：自动生成实验方案，一键在本地 GPU 执行
  - 学者匹配：AI 发现跨学科合作机会
- 底部：简单的 footer

### 2. 论文搜索页 `/search`

**目标**: 快速找到相关论文，是进入深度研究的入口。

**布局**:
- 顶部搜索栏（sticky）：
  - 搜索类型下拉：关键词 / 标题 / 作者 / DOI
  - 搜索输入框（占满剩余宽度）
  - 搜索按钮
- 左侧筛选栏（可折叠，240px）：
  - 年份范围（滑块，如 2015-2026）
  - 引用数范围（滑块）
  - 数据源勾选（OpenAlex / Semantic Scholar / PubMed / arXiv / CNKI / 万方）
  - 语言（中文 / 英文 / 全部）
  - Open Access 开关
- 主内容区：
  - 排序栏：相关性 / 引用数 / 年份（最新）
  - 结果计数 + 来源信息
  - 论文卡片列表，每张卡片包含：
    - 标题（可点击，链接到论文详情或外部 DOI）
    - 作者列表（超过 5 人折叠）
    - 年份 | 期刊/会议 | 引用数
    - 摘要（3 行截断，可展开）
    - 标签：OA 标识、来源标识（arXiv、CNKI 等）
    - 操作：加入阅读列表、开始深度研究、查看引用图谱
  - 底部分页或无限滚动

### 3. 论文地图页 `/research/[taskId]`

**目标**: 平台核心页面。让研究者一眼看清整个研究领域的论文版图。

**布局**: 全屏画布 + 侧边面板

- 顶部工具栏（浮动）：
  - 视图切换标签：引用图谱 / 主题地图 / 时间线
  - 筛选控件：时间范围、方法类型、质量阈值
  - 导出按钮
  - 保存阅读列表按钮
- 主画布区（占满屏幕）：
  - **引用图谱视图**（默认）：
    - React Flow 力导向布局
    - 节点 = 论文，大小 ∝ 引用数
    - 节点颜色 = 研究簇/方法家族（自动聚类着色）
    - 边 = 引用关系，有方向箭头
    - hover 显示论文标题 + 年份
    - 支持缩放、平移、框选、拖拽节点
    - 双击节点可展开其引用网络
  - **主题地图视图**：
    - Deck.gl 散点图，类似 O-DataMap
    - 每个点 = 一篇论文，位置由论文嵌入向量降维决定
    - 聚类区域半透明着色 + 标签（如"Transformer 架构"、"强化学习"、"图神经网络"）
    - 点大小 ∝ 质量分或引用数
    - hover/click 查看论文详情
  - **时间线视图**：
    - 横轴 = 时间，纵轴 = 研究子方向
    - 关键里程碑论文突出显示
    - 可按年份缩放
- 右侧详情面板（320px，点击节点时滑出）：
  - 论文标题、作者、年份、期刊
  - 质量分（总分 + 各维度分解）
  - 摘要
  - AI 分析：使用的方法、关键发现
  - 研究空白（如果 AI 检测到）
  - 操作：打开原文、加入阅读列表、基于此论文生成实验方案

### 4. 深度研究（集成在论文地图流程中）

**目标**: 用户发起深度研究 → 等待 AI 处理 → 自动跳转到论文地图。

**入口**: 首页三种研究入口 / 搜索页的"开始深度研究"按钮

**多轮搜索机制**:
- 系统根据用户输入（方向/论文/对话结果）自动拆分关键词
- 第一轮：核心关键词搜索，获取初始论文集
- 后续轮次：基于已找到论文的关键词、引用关系、AI 分析结果，自动扩展搜索词
- 每轮搜索结果去重、评分、筛选，逐步构建完整的论文网络
- 用户可在进度页面看到每轮搜索使用的关键词和新发现的论文数

**流程页面**:
- Step 1 - 配置：
  - 研究方向输入（必填，从首页入口自动填充）
  - 搜索深度（1-3 层引用扩展）
  - 搜索轮次上限（默认 5 轮）
  - 数据源勾选
  - 时间范围
  - 语言偏好
  - "开始研究"按钮
- Step 2 - 进度（WebSocket 实时更新）：
  - 管道阶段可视化（横向步骤条）：搜索论文 → 扩展引用 → 质量评分 → AI 分析 → 分类关系 → 检测空白 → 生成报告
  - 当前阶段高亮 + 动画
  - 实时数字：已找到论文数、已分析论文数、预估剩余时间
  - 日志流（可折叠，显示详细处理日志）
- Step 3 - 完成：
  - 自动跳转到论文地图页
  - 或查看文献综述报告（Markdown 渲染）

**目标**: 查看 AI 生成的实验改进方案，选择执行。

  - 代码骨架预览（语法高亮代码块）
  - 可行性评分分解：计算需求、数据可用性、预期提升、实现难度

### 6. 实验仪表盘 `/experiments`

**目标**: 监控实验进度，查看结果。

**布局**:
- 左侧实验列表（280px）：
  - 运行中 / 排队中 / 已完成 分组
  - 每项：实验名、状态图标、进度百分比
  - 拖拽排序队列
- 主内容区（选中实验的详情）：
  - 概览卡片行：当前轮次、最佳指标值、相比基线提升百分比、运行时间

    - 横轴 = 迭代轮次
    - 多条线 = 不同指标（loss、accuracy、F1 等）

  - 迭代历史表格：
    - 列：轮次、代码修改摘要、指标值、是否保留、时间

  - GPU 监控面板（实时刷新）：
    - GPU 利用率仪表盘
    - 显存使用柱状图
    - 温度折线
  - 操作栏：暂停 / 恢复 / 跳过当前迭代 / 取消 / 手动指导（输入建议）
- 实验报告页（完成后可查看）：

  - 包含自动生成的图表

## 种子数据（研究方向预置）

平台需要预置一批研究方向作为种子数据，每个方向包含：

| 字段 | 说明 |
|------|------|
| 方向名称 | 如"自然语言处理"、"计算机视觉"、"强化学习" |
| 子方向 | 如 NLP 下的"大语言模型"、"信息抽取"、"机器翻译" |
| 种子关键词 | 该方向的核心搜索词（中英文各 5-10 个） |
| 种子论文 | 该方向的 3-5 篇里程碑论文（DOI） |
| 描述 | 一句话介绍该方向 |

初始种子方向建议覆盖：
- 计算机科学：NLP、CV、强化学习、图神经网络、推荐系统、自动驾驶
- AI 交叉：AI+医疗、AI+材料、AI+金融、AI+教育、AI+法律
- 基础科学：量子计算、生物信息学、气候建模

**用户自定义方向**：
- 用户可以直接输入自定义研究方向（不局限于预置列表）
- 输入自定义方向后，AI 自动拆分为关键词并启动搜索
- 系统根据搜索结果自动生成该方向的种子关键词和代表性论文
- 用户的自定义方向保存到个人收藏，后续可复用

**自动搜索流程**：
- 用户选择预置方向或输入自定义方向后，系统立即开始自动搜索
- 无需额外配置即可一键启动（高级配置可选展开）
- 搜索完成后自动跳转到论文地图页

种子数据存储在后端数据库，前端通过 API 获取（`GET /api/v1/directions/`），支持管理员后台增删改。

## 设计要求

1. **现代简洁**: 大量留白，信息层次清晰，不堆砌功能
2. **数据可视化优先**: 图谱、图表是核心卖点，要流畅美观
3. **中文优先**: 默认 zh-CN，所有文案先写中文
4. **响应式**: 支持桌面（1920px）和平板（768px），不需要手机适配
5. **暗色/亮色**: 初期只做亮色主题
6. **无障碍**: 按钮有 hover/focus 状态，表单有 label，图片有 alt
