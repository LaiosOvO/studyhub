2. Create `backend/app/services/plan_generation/prompts.py` with ALL prompt builders for the entire pipeline (Plans 01 and 02 will use these):
   - `build_sota_prompt(direction, paper_count, top_methods, best_metrics, hf_benchmarks)` -> list[dict]: System + user message. Instructs LLM to identify SOTA methods, standard baselines, evaluation metrics, benchmark datasets. Returns JSON format matching SOTAResult schema. Based on SOTA_IDENTIFICATION_PROMPT from research.
   - `build_improvement_prompt(direction, sota_summary, gaps_summary, entry_context)` -> list[dict]: Analyzes gaps + SOTA to produce improvement opportunities. Returns JSON list of {gap_description, improvement_type, suggested_approach, estimated_difficulty, related_gap_index}.
   - `build_plan_generation_prompt(direction, sota_summary, gaps_summary, improvements_summary, entry_context)` -> list[dict]: Generates experiment plan. Returns JSON matching plan fields (title, hypothesis, method_description, baselines, metrics, datasets, technical_roadmap, interestingness, feasibility, novelty). Based on PLAN_GENERATION_PROMPT from research.
   - `build_reflection_prompt(plan_draft_json, round_num, total_rounds)` -> list[dict]: AI-Scientist-style reflection. Asks LLM to critique and improve the plan. Instructs to return "I am done" if converged, otherwise return improved JSON in same format.
   - `build_feasibility_prompt(plan_summary, direction)` -> list[dict]: Scores compute_requirements (1-5), data_availability (1-5), expected_improvement (1-5), difficulty (1-5) with explanation. For Haiku (cost efficiency).
   - All prompts end with "Return ONLY valid JSON, no additional text."
   - All prompts are bilingual-aware (mention that direction may be in Chinese).

3. Create `backend/app/services/plan_generation/sota_identifier.py`:
   - `aggregate_methods_by_metric(paper_analyses: dict[str, dict]) -> dict[str, list[dict]]`: Pure function. Groups methods by their associated metrics from PaperAnalysis data. Returns {metric_name: [{method, value, paper_id}]}.
   - `async def search_hf_benchmarks(direction: str, limit: int = 5) -> str`: Uses `huggingface_hub.HfApi().list_datasets(search=keyword, sort="downloads", direction=-1, limit=limit)`. Wraps in `asyncio.to_thread` (HfApi is sync, same pattern as pyalex). Extracts top datasets matching the direction. Returns formatted string for prompt context. Non-fatal: returns "No external benchmarks available" on failure.
   - `async def identify_sota(task: DeepResearchTask, session: AsyncSession) -> SOTAResult`: Main entry point.
     Step 1: Extract paper_analyses from task.config, call aggregate_methods_by_metric.
     Step 2: Call search_hf_benchmarks with task.research_direction.
     Step 3: Build prompt with build_sota_prompt, call llm_completion (Sonnet, request_type="plan_sota").
     Step 4: Parse JSON response into SOTAResult. Return empty SOTAResult on failure.
   - Import `from app.schemas.plan import SOTAResult, SOTAMethod`

4. Create `backend/app/services/plan_generation/improvement_analyzer.py`:
   - `async def create_plan_context(entry_type: str, task: DeepResearchTask, session: AsyncSession, paper_id: str | None = None, gap_index: int | None = None) -> PlanGenerationContext`: Three-entry-point dispatcher per PLAN-08.
     - "direction": full SOTA + all gaps + trends + top papers (by quality score)
     - "paper": focus paper loaded from DB + SOTA + paper-specific improvements
     - "gap": specific gap extracted by index + SOTA + related papers
     All paths call identify_sota internally.
   - `async def analyze_improvements(sota: SOTAResult, gaps: GapResult, direction: str, session: AsyncSession, user_id: str) -> list[ImprovementOpportunity]`: Uses build_improvement_prompt + llm_completion (Sonnet, request_type="plan_improvements"). Parses JSON response. Returns empty list on failure.
   - `def get_top_papers(task: DeepResearchTask, limit: int = 10) -> list[dict]`: Pure function. Extracts papers with highest quality metrics from paper_analyses config. Returns list of dicts with paper_id, title, methods, key_metrics.
