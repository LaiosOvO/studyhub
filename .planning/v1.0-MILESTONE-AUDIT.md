## Critical Blockers (3)

### 1. (auth) Route Group Layout Inverts Auth Guard

**File:** `apps/web/src/app/[locale]/(auth)/layout.tsx`

The `(auth)` route group layout redirects authenticated users to `/` when `isAuthenticated === true`. This group wraps login/register pages (correct) AND experiment, community, and research/map pages (incorrect). Every protected page is unreachable for logged-in users.

**Impact:** PMAP-01..10, DASH-01..05, REPT-01..04, PROF-01..06, MTCH-01..04, NEED-01..04, MESG-01..03 (49 requirements)

**Fix:** Move protected pages to a new `(protected)` route group with a layout that redirects unauthenticated users to login. Keep `(auth)` only for login/register.

### 2. Temporal Worker Missing Activity Registrations

**File:** `backend/app/worker.py`

Worker registers only 4 activities (placeholder_search, enrich_scholar, link_scholar_papers, get_all_scholar_ids). Missing: all 7 deep research activities + generate_plans_activity.

**Impact:** DEEP-01..08, ANAL-01..06, PLAN-01..08 (22 requirements)

**Fix:** Import and register all activity functions from `app.workflows.activities` and add `PlanGenerationWorkflow` to the workflows list.

### 3. No Frontend Search UI

**File:** None exists

Backend search router with multi-source aggregation and Meilisearch exists but has zero frontend consumers.

**Impact:** SRCH-01..08 (8 requirements)

**Fix:** Create search page with search bar, results list, filters, and sorting. This is a core user flow entry point.
