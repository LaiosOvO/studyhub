    **`backend/app/services/experiment/gpu_monitor.py`**: GPU metrics collection using nvidia-ml-py (already added in Plan 08-02). Reference: 08-RESEARCH.md GPU Metrics Collection example.

    ```python
    @dataclass(frozen=True)
    class GpuInfo:
        index: int
        name: str
        memory_total_mb: int
        driver_version: str

    def list_gpus() -> list[GpuInfo]:
        """Enumerate available GPUs. Returns empty list if no NVIDIA GPU or driver."""

    def get_gpu_metrics(device_index: int = 0) -> dict:
        """Collect GPU metrics for real-time display.
        Returns: {name, gpu_utilization_pct, memory_used_mb, memory_total_mb, temperature_c, power_watts}
        If pynvml fails (no GPU, driver issue): returns dict with all zeros and name="unavailable".
        """

    def gpu_available() -> bool:
        """Quick check: is pynvml usable? Returns False on any exception."""
    ```

    All functions are synchronous (pynvml is sync). Use try/except around all pynvml calls -- GPU monitoring must never crash the app.

    The gpu_monitor.py can also be run as a standalone script for Tauri shell plugin:
    ```python
    if __name__ == "__main__":
        import json, sys, time
        device = int(sys.argv[1]) if len(sys.argv) > 1 else 0
        interval = float(sys.argv[2]) if len(sys.argv) > 2 else 1.0
        while True:
            print(json.dumps(get_gpu_metrics(device)), flush=True)
            time.sleep(interval)
    ```

    **`apps/desktop/src-tauri/src/commands/gpu.rs`**: Tauri commands for GPU monitoring. Reference: 08-RESEARCH.md Pattern 4 (event streaming).

    - `get_gpu_info() -> Result<serde_json::Value, String>`: Run Python gpu_monitor.py script via shell plugin (or subprocess), parse JSON output. One-shot call returning list of GPUs.

    - `start_gpu_monitoring(device_id: u32, app: AppHandle) -> Result<(), String>`:
      Spawn background task that:
      1. Runs gpu_monitor.py as subprocess with device_id and interval args
      2. Reads stdout line-by-line (each line is JSON metrics)
      3. Emits "gpu-metrics" Tauri event with parsed metrics
      4. Stores task handle in a monitoring state for cancellation

    - `stop_gpu_monitoring(state: State<GpuMonitorState>) -> Result<(), String>`: Cancel the monitoring background task.

    Define `GpuMonitorState` with Mutex<Option<JoinHandle>> for tracking the monitoring task.

    **`apps/desktop/src-tauri/src/commands/sync.rs`**: WebSocket sync with web backend. Reference: 08-RESEARCH.md Pattern 4.

    - `connect_backend(url: String, token: String, state: State<SyncState>) -> Result<(), String>`:
      Connect to web backend WebSocket at `{url}/ws/experiment-sync?token={token}`.
      Store connection in SyncState.

    - `disconnect_backend(state: State<SyncState>) -> Result<(), String>`: Close WebSocket connection.

    - `send_sync(payload: serde_json::Value, state: State<SyncState>) -> Result<(), String>`:
      Send ExperimentSyncPayload JSON to web backend via WebSocket.

    Define `SyncState` with Mutex for WebSocket connection handle.

    **Update `commands/mod.rs`**: Add `pub mod gpu;` and `pub mod sync;` re-exports.

    **Update `lib.rs`**: Register GpuMonitorState and SyncState as managed state. Add GPU and sync commands to the invoke_handler.
