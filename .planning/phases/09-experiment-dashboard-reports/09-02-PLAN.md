  <action>
    **1. Queue reorder schema** (`backend/app/schemas/experiment.py`):
    Add `ExperimentQueueReorder` schema:
    ```python
    class ExperimentQueueReorder(BaseModel):
        after_run_id: str | None = None   # Place after this run (None = first)
        before_run_id: str | None = None  # Place before this run (None = last)
    ```

    **2. Queue reorder endpoint** (`backend/app/routers/experiments.py`):
    Add `PATCH /{run_id}/reorder` endpoint:
    - Accept `ExperimentQueueReorder` body.
    - Only allow reordering of runs with status "pending".
    - Compute new `queue_position` using fractional positioning:
      - If `after_run_id` provided, get its queue_position as `after_pos`.
      - If `before_run_id` provided, get its queue_position as `before_pos`.
      - If neither provided, set to max(all positions) + 1.0.
      - If only `after_run_id`, query next pending run after it to find `before_pos`. If none, use `after_pos + 1.0`.
      - If only `before_run_id`, query previous pending run before it to find `after_pos`. If none, use `before_pos - 1.0`.
      - new_pos = (after_pos + before_pos) / 2.0
    - Update run's queue_position with single UPDATE query (atomic, no read-modify-write race).
    - Return updated ExperimentRunResponse.
    - Validate ownership of all referenced runs.

    **3. Add cancel endpoint** (`backend/app/routers/experiments.py`):
    Add `POST /{run_id}/cancel` endpoint:
    - Validates transition (only pending/running/paused can cancel).
    - Sets status to "cancelled", completed_at to now.
    - Returns updated ExperimentRunResponse.

    **4. API client additions** (`apps/web/src/lib/api/experiments.ts`):
    Add functions:
    - `reorderExperimentRun(runId: string, afterRunId?: string, beforeRunId?: string): Promise<ExperimentRun>` — calls PATCH /{run_id}/reorder
    - `cancelExperimentRun(runId: string): Promise<ExperimentRun>` — calls POST /{run_id}/cancel

    **5. IterationTable component** (`apps/web/src/components/experiments/IterationTable.tsx`):
    'use client' component. Props: `{ rounds: readonly ExperimentRoundResult[] }` (readonly).

    - Sort state: `useState<{ column: string; direction: 'asc' | 'desc' }>`. Default: round ascending.
    - Column headers are clickable to toggle sort. Show sort indicator arrow.
    - Create sorted copy of rounds (immutable — never mutate the prop).
    - Status badges: green for "keep"/"baseline", red for "crash", yellow for "discard".
    - Empty state: show "No iterations yet" message.
    - Use next-intl for all labels.
