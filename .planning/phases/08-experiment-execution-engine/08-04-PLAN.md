    **`backend/app/services/experiment/loop_agent.py`**: The autonomous experiment loop. Reference: autoresearch program.md loop + AI-Scientist perform_experiments.py.

    ```python
    @dataclass(frozen=True)
    class ExperimentConfig:
        plan_id: str
        run_id: str
        workspace: Path
        gpu_device: int = 0
        max_rounds: int = 20
        consecutive_no_improve_limit: int = 5
        time_budget_minutes: int | None = None
        primary_metric: str = "val_loss"
        lower_is_better: bool = True
        max_fix_attempts: int = 3

    class ControlSignal(Enum):
        CONTINUE = "continue"
        PAUSE = "pause"
        RESUME = "resume"
        SKIP = "skip"
        CANCEL = "cancel"
        GUIDE = "guide"  # user_guidance attached

    async def run_experiment_loop(
        config: ExperimentConfig,
        runner: DockerRunner,
        git: GitManager,
        control_queue: asyncio.Queue,  # receives ControlSignal from Tauri
        on_iteration: Callable,  # callback(round_num, result_dict)
        user_guidance: str | None = None,
    ) -> dict:
    ```

    Loop logic (EXPR-04):
    1. Read current train.py and results history
    2. Check control_queue for pause/skip/cancel/guide signals (EXPR-10)
       - PAUSE: await queue.get() until RESUME received
       - SKIP: continue to next iteration
       - CANCEL: break loop, return results
       - GUIDE: set user_guidance for next prompt (EXPR-07)
    3. Build LLM prompt (use build_guidance_prompt if user_guidance set, else build_analysis_prompt)
    4. Call LLM for analysis -> get proposed change description
    5. Call LLM for code modification -> get modified train.py
    6. Write modified train.py to workspace
    7. Git commit: "experiment {round}: {change_description}"
    8. Run training via DockerRunner
    9. Parse output metrics
    10. If crash: try fix loop (up to max_fix_attempts) -- call build_fix_prompt, rewrite, rerun
    11. If still crash after fixes: git reset, record as "crash", increment round
    12. If metric improved (respecting lower_is_better): keep, update best_metric
    13. If metric did not improve: git reset_to_previous, record as "discard"
    14. Append to results.tsv
    15. Call on_iteration callback
    16. Check stopping conditions (EXPR-06):
        - round >= max_rounds
        - consecutive_no_improve >= consecutive_no_improve_limit
        - elapsed time >= time_budget_minutes
    17. Return final results dict: {status, total_rounds, best_metric, improvement_over_baseline, rounds: [...]}

    Important patterns:
    - NEVER mutate config -- it's frozen dataclass
    - Git stash_and_clean before each iteration for safety
    - All LLM calls use existing llm_completion from llm_service.py with Sonnet model
    - on_iteration callback enables Tauri to emit events and sync to web
    - user_guidance is consumed once then reset to None
