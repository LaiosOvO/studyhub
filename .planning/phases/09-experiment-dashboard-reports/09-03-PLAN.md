    Pure functions for generating chart PNGs from experiment data. Reference: AI-Scientist figure generation pattern.

    ```python
    import matplotlib
    matplotlib.use("Agg")  # Headless backend — MUST be before pyplot import
    import matplotlib.pyplot as plt
    ```

    Functions:
    - `generate_training_curve_png(rounds: list[dict], metric_name: str, output_path: Path) -> Path`:
      - Filter rounds where metric_value is not None.
      - x = round numbers, y = metric values.
      - Plot line chart with markers colored by status (green="keep"/"baseline", red="crash", orange="discard").
      - Highlight baseline value as horizontal dashed line if present.
      - Title: f"Training Curve: {metric_name}", xlabel="Round", ylabel=metric_name.
      - Grid enabled, tight_layout, save at dpi=150.
      - CRITICAL: Use `try/finally` with `plt.close(fig)` to prevent memory leaks.
      - Return output_path.

    - `generate_comparison_chart_png(rounds: list[dict], metric_name: str, output_path: Path) -> Path`:
      - Bar chart comparing metric values across rounds.
      - Bars colored: green for "keep", red for "crash"/"discard", blue for "baseline".
      - Only include rounds with metric_value (skip crashes without metrics).
      - xlabel="Round", ylabel=metric_name, title="Metric Comparison Across Rounds".
      - Same plt.close(fig) pattern.
      - Return output_path.

    - `generate_improvement_chart_png(rounds: list[dict], baseline_value: float | None, metric_name: str, output_path: Path) -> Path`:
      - Line chart showing percentage improvement over baseline per round.
      - If baseline_value is None, return None (can't compute improvement).
      - y = ((baseline - metric) / |baseline|) * 100 for each round with metric_value.
      - Horizontal line at 0% (baseline reference).
      - Green fill above 0, red fill below.
      - Return output_path.

    All functions are pure (no side effects beyond writing the PNG file). All use immutable input patterns.

    3. Use `summarize_results` from metrics.py to compute summary stats from `run.rounds`.

       - Import `llm_complete` from `app.services.llm_gateway` (existing).

       - Prompt for conclusion: "Given the above results, write a conclusion summarizing findings and suggesting next steps (150 words)."
       - Use Haiku model for cost efficiency (same pattern as Phase 5).
       - If LLM fails, use fallback text: "Abstract generation failed. Please write manually." / "结论生成失败，请手动撰写。"
    5. Generate ablation analysis prompt if there are 5+ rounds with varied approaches (optional, empty string if not enough data).
    6. Render Jinja2 template with context: plan_title, generated_date, plan.hypothesis, plan.method_description, plan.baselines, rounds, summary_stats, abstract, conclusion, ablation, chart image filenames, language.
    7. Convert Markdown to PDF via `markdown_to_pdf` helper function.

    Helper function:
    ```python
    def markdown_to_pdf(md_content: str, image_dir: Path) -> bytes:
    ```

    - Convert Markdown to HTML using `markdown.markdown(md_content, extensions=["tables", "fenced_code"])`.
