**Create** `backend/app/services/deep_research/gap_detector.py`:

1. `def build_corpus_summary(analyses: list[PaperAnalysis]) -> str`:
   - Pure function. Aggregate methods (Counter), datasets (Counter), paper_types (Counter).
   - Format as readable text: "Methods used: X (N papers), Y (M papers)..."
   - Truncate to 3000 chars to fit LLM context.

2. `def build_method_frequencies(analyses: list[PaperAnalysis], papers: list[Paper]) -> str`:
   - Pure function. Group methods by paper year. Format as "2020: method_a (5), method_b (3)\n2021: ..."
   - Used for both gap detection and trend analysis.

3. `GapResult` Pydantic model: gaps (list of {description, evidence, potential_impact}), underexplored (list of {combination, why_promising}), missing_evaluations (list of {method, missing}).

4. `async def detect_gaps(analyses: list[PaperAnalysis], papers: list[Paper], direction: str, session: AsyncSession, user_id: str) -> GapResult`:
   - Build corpus_summary and method_frequencies.
   - Call llm_completion with GAP_DETECTION_PROMPT, model=None (Sonnet for quality), max_tokens=2048, request_type="deep_research_gaps".
   - Parse JSON into GapResult. Handle JSONDecodeError with empty GapResult.

5. `TrendResult` Pydantic model: ascending_methods (list of {method, evidence}), declining_methods (list), emerging_topics (list), stable_methods (list).

6. `async def detect_trends(analyses: list[PaperAnalysis], papers: list[Paper], direction: str, session: AsyncSession, user_id: str) -> TrendResult`:
   - Build method_frequencies grouped by year.
   - Call llm_completion with TREND_INTERPRETATION_PROMPT, model="claude-haiku-4-20250514" (trends are simpler), max_tokens=1024, request_type="deep_research_trends".
   - Parse JSON into TrendResult.

**Create** `backend/app/services/deep_research/report_generator.py`:

1. `async def generate_literature_review(task: DeepResearchTask, analyses: list[PaperAnalysis], gaps: GapResult, trends: TrendResult, papers: list[Paper], language: str = "zh") -> str`:
   - Load Jinja2 template from `backend/templates/literature_review.md.j2`.
   - Build template context: direction, paper_count, top_papers (sorted by quality, top 10 with TLDRs), method_summary, gaps, trends, relationship_summary.
   - Render template. Return Markdown string.
   - Use `jinja2.Environment(loader=FileSystemLoader("templates"), autoescape=False)` since output is Markdown.

**Create** `backend/templates/literature_review.md.j2`:
Bilingual Jinja2 template with sections:
- Title: "Literature Review: {direction}" / "文献综述: {direction}"
- Overview: paper count, date range, sources
- Key Papers: Top 10 by quality with TLDR (use tldr_zh if language=="zh" else tldr_en)
- Methodology Landscape: method frequency table
- Research Gaps: from GapResult.gaps (description + evidence + impact)
- Trends: ascending/declining methods with evidence
- Underexplored Combinations: from GapResult.underexplored
- References: numbered list of all papers (author, title, year, venue)

Template uses Jinja2 `{% if language == "zh" %}` blocks for bilingual support. Default to Chinese.

Install jinja2: `uv add jinja2` (per research doc).
