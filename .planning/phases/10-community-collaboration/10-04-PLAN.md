    **1. Message service** (`backend/app/services/community/message_service.py`):
    - `async def send_message(session: AsyncSession, valkey: Valkey, sender_id: str, recipient_id: str, content: str) -> Message`:
      - Validate content not empty, sender != recipient.
      - Create Message record in DB.
      - Publish to Valkey channel `user:{recipient_id}:messages` with JSON payload: `{id, sender_id, content, created_at}`.
      - Increment unread counter: `await valkey.incr(f"unread:{recipient_id}")`.
      - Return the new Message. All Valkey operations non-fatal (try/except, log warning).
    - `async def get_conversation(session: AsyncSession, user_id: str, other_user_id: str, skip: int = 0, limit: int = 50) -> list[Message]`:
      - Query messages where (sender_id=user_id AND recipient_id=other_user_id) OR (sender_id=other_user_id AND recipient_id=user_id).
      - Order by created_at DESC. Apply skip/limit.
      - Return list of Message objects.
    - `async def mark_conversation_read(session: AsyncSession, valkey: Valkey, user_id: str, other_user_id: str) -> int`:
      - UPDATE messages SET read_at=now() WHERE recipient_id=user_id AND sender_id=other_user_id AND read_at IS NULL.
      - Recalculate total unread count: SELECT COUNT(*) FROM messages WHERE recipient_id=user_id AND read_at IS NULL.
      - Update Valkey: `await valkey.set(f"unread:{user_id}", total_unread)`.
      - Return count of messages marked as read.
    - `async def get_conversations(session: AsyncSession, user_id: str) -> list[dict]`:
      - SQL: Get distinct conversation partners with latest message and unread count per partner.
      - Use a subquery with ROW_NUMBER() OVER (PARTITION BY partner ORDER BY created_at DESC) to get latest message per partner.
      - Also join User table to get partner's full_name.
      - Return list of dicts matching ConversationListItem schema.

    **2. Notification service** (`backend/app/services/community/notification_service.py`):
    - `async def get_unread_count(valkey: Valkey, user_id: str) -> int`:
      - `count = await valkey.get(f"unread:{user_id}")`.
      - Return int(count) if count else 0.
    - `async def mark_all_read(valkey: Valkey, user_id: str) -> None`:
      - `await valkey.set(f"unread:{user_id}", 0)`.
    - `async def sync_unread_count(session: AsyncSession, valkey: Valkey, user_id: str) -> int`:
      - Count unread messages from DB: `SELECT COUNT(*) FROM messages WHERE recipient_id=user_id AND read_at IS NULL`.
      - Update Valkey counter to match DB truth.
      - Return count.

    **3. Messages router** (`backend/app/routers/messages.py`):
    - `router = APIRouter(prefix="/messages", tags=["messages"])`
    - `POST /` (auth required): Send message. Call `send_message(session, valkey, user.id, data.recipient_id, data.content)`. Return 201 with MessageResponse.
    - `GET /conversations` (auth required): List conversations. Call `get_conversations(session, user.id)`. Return list of ConversationListItem.
    - `GET /conversations/{other_user_id}` (auth required): Get conversation with specific user. Call `get_conversation(session, user.id, other_user_id, skip, limit)`. Return list of MessageResponse.
    - `POST /conversations/{other_user_id}/read` (auth required): Mark conversation as read. Call `mark_conversation_read(session, valkey, user.id, other_user_id)`. Return `{marked_read: int}`.
    - `GET /unread` (auth required): Get unread count. Call `get_unread_count(valkey, user.id)`. Return `{unread_count: int}`.
    - `WebSocket /ws` (auth via query param token, same pattern as experiments WebSocket):
      - Verify JWT from `token` query param.
      - Accept WebSocket.
      - Create Valkey pubsub, subscribe to `user:{user_id}:messages`.
      - Listen loop: forward messages to WebSocket client.
      - On disconnect, unsubscribe.
      - Fall back to 5s polling of unread count if Valkey unavailable.

    **4. Register router** in `routers/__init__.py`.

    NOTE: Use `from app.dependencies import get_db, get_current_user, get_valkey` for injection.
    NOTE: WebSocket auth follows the same pattern as experiments.py -- JWT token in query param.
    NOTE: All database queries create new objects, never mutate.
