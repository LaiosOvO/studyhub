    **1. Need matcher** (`backend/app/services/community/need_matcher.py`):
    - `def score_need_match(viewer_expertise: list[str], need_required_skills: list[str]) -> float`:
      Reuse the complementarity concept but for direct skill match (not inverted).
      Combine viewer's expertise_tags + research_directions as their "skills".
      Compute TF-IDF similarity between viewer skills and need's required_skills.
      Return raw cosine similarity (0-1) -- for needs, high similarity = good match (you have what they need).
      If either list empty, return 0.0.
    - `def score_direction_match(viewer_directions: list[str], need_direction: str | None) -> float`:
      If need has no research_direction, return 0.0.
      Check if any of viewer's directions fuzzy-match (rapidfuzz ratio >= 70) the need's direction.
      Return 1.0 if match, 0.0 if not.
    - `def compute_need_relevance(viewer_profile: ResearcherProfile, need: ResearchNeed) -> float`:
      Weighted: 70% skill match + 30% direction match.
      Return float 0-1.

    **2. Needs CRUD router** (`backend/app/routers/needs.py`):
    - `router = APIRouter(prefix="/needs", tags=["needs"])`
    - `POST /` (auth required): Create ResearchNeed for current user. Index in Meilisearch (index name "research_needs", searchable attributes: title, description, required_skills, research_direction, tags; filterable: tags, research_direction, status). Return 201.
    - `GET /` (auth required): Browse needs. Query params: q (search text), tags (comma-separated), research_direction, status (default "open"), sort_by ("relevance" or "recent"), skip, limit.
      - If q provided, search Meilisearch first, then fall back to DB LIKE query.
      - If sort_by="relevance" and viewer has a profile, compute `compute_need_relevance` for each result and sort by relevance descending.
      - Return list of ResearchNeedResponse with added `match_score: float` field (add to schema).
    - `GET /{need_id}` (auth required): Get single need.
    - `PATCH /{need_id}` (auth required): Update own need. 403 if not owner.
    - `DELETE /{need_id}` (auth required): Delete own need. 403 if not owner. Remove from Meilisearch.
    - `POST /{need_id}/close` (auth required): Set status to "closed". Owner only.

    **3. Register router** in `routers/__init__.py`.

    **4. Add `match_score` to need response**: Add `ResearchNeedWithScore(ResearchNeedResponse)` schema with `match_score: float = 0.0` in `schemas/need.py`.

    NOTE: Meilisearch index creation should be idempotent (check if exists first, same pattern as papers index).
    NOTE: Meilisearch failure is non-fatal -- fall back to PostgreSQL queries.
