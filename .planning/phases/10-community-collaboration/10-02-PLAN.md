    **1. Matching engine** (`backend/app/services/community/matching_engine.py`):

    Pure functions for each matching signal:

    a) `compute_complementarity(keywords_a: list[str], keywords_b: list[str]) -> float`:
    - Combine all keywords into a corpus of two "documents" (space-joined keyword lists).
    - Use `TfidfVectorizer().fit_transform(corpus)` to get TF-IDF vectors.
    - Compute `raw_sim = cosine_similarity(vec_a, vec_b)[0][0]`.
    - Complementarity bell curve: `math.exp(-((raw_sim - 0.35) ** 2) / (2 * 0.15 ** 2))`.
    - Return float 0-1. If either keyword list is empty, fall back to 0.3 (neutral).

    b) `compute_adjacency(directions_a: list[str], directions_b: list[str]) -> float`:
    - Jaccard coefficient on research directions: `|A ∩ B| / |A ∪ B|`.
    - Optimal adjacency: some overlap but not full overlap. Score = `1 - abs(jaccard - 0.3) / 0.7` (peaks at ~30% overlap).
    - Return float 0-1. If either list empty, return 0.0.

    c) `compute_institutional_proximity(inst_a: str | None, inst_b: str | None) -> float`:
    - If both None, return 0.0.
    - If exact match (case-insensitive), return 1.0.
    - Use `rapidfuzz.fuzz.ratio(inst_a, inst_b) / 100.0` for fuzzy match.
    - Return float 0-1.

    d) `async def compute_co_citation(profile_a_papers: list[str], profile_b_papers: list[str], neo4j_client) -> float`:
    - If either has no papers or neo4j_client is None, return 0.0.
    - Query Neo4j: papers from A that cite or are cited by papers from B.
    - Cypher: `MATCH (a:Paper)-[:CITES]-(shared:Paper)-[:CITES]-(b:Paper) WHERE a.id IN $a_ids AND b.id IN $b_ids RETURN count(DISTINCT shared) AS overlap`.
    - Normalize: `min(overlap / 10.0, 1.0)`.
    - Wrap in try/except (non-fatal if Neo4j unavailable).

    e) `MATCH_WEIGHTS` constant dict: `{"complementarity": 0.40, "co_citation": 0.25, "adjacency": 0.20, "institutional": 0.15}`.

    f) `compute_match_score(signals: dict[str, float]) -> float`: Weighted sum.

    g) `async def find_matches(profile: ResearcherProfile, candidates: list[ResearcherProfile], neo4j_client=None, top_n: int = 20) -> list[dict]`:
    - For each candidate (excluding self by user_id), compute all four signals.
    - Optimization: compute cheap signals first (complementarity, adjacency, institutional). Only compute co_citation for candidates scoring > 0.3 on cheap signals.
    - Sort by overall_score descending. Return top_n as list of dicts: `{profile_id, overall_score, breakdown: {complementarity, co_citation, adjacency, institutional}}`.
    - All computation creates new dicts, no mutation.

    **2. Match explainer** (`backend/app/services/community/match_explainer.py`):
    - `async def generate_explanation(profile_a: ResearcherProfilePublic, profile_b: ResearcherProfilePublic, breakdown: MatchSignalBreakdown) -> str`:
    - Build prompt: "Given two researchers: Researcher A ({name}, {institution}, fields: {directions}, expertise: {tags}) and Researcher B ({name}, {institution}, fields: {directions}, expertise: {tags}). Match scores: complementarity={X}, co-citation={Y}, adjacency={Z}. In 2-3 sentences, explain why they would be good collaborators. Respond in the user's language."
    - Call `llm_completion(prompt, model="claude-haiku-4-20250514")`.
    - Return explanation string. On error, return a generic message: "These researchers have complementary expertise."

    **3. Match router** (`backend/app/routers/matching.py`):
    - `router = APIRouter(prefix="/matching", tags=["matching"])`
    - `GET /recommendations` (auth required):
      - Get current user's profile. 404 if no profile exists.
      - Check Valkey cache: `matches:{user_id}`. If exists and < 1 hour old, return cached.
      - Fetch all other profiles from DB (limit 500 for v1).
      - Call `find_matches(profile, candidates, neo4j_client, top_n=20)`.
      - For top-5 matches, generate LLM explanations (await all in parallel via `asyncio.gather`).
      - For matches 6-20, set explanation=None (lazy — generated on detail view).
      - Cache results in Valkey (TTL=3600s).
      - Return list of MatchResult.
    - `GET /recommendations/{profile_id}/explain` (auth required):
      - Generate explanation for a specific match pair if not already cached.
      - Return `{explanation: str}`.

    Register router in `routers/__init__.py`.

    NOTE: Neo4j client is optional. If unavailable, co_citation signal returns 0.0 and weight redistributes to other signals.
    NOTE: Use `from app.services.llm_service import llm_completion` for LLM calls.
