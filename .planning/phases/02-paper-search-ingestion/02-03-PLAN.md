Build the PDF parsing pipeline: GROBID Docker service, PDF-to-structured-sections extraction, and storage of parsed content for LLM analysis.

Purpose: Full-text parsing enables downstream AI features (summarization, methodology extraction, gap identification). Without structured sections, the LLM can only work with abstracts.
Output: GROBID running in Docker, parse endpoint that extracts structured sections from PDFs and stores them in the papers table.
</objective>

    **Create backend/app/services/pdf_parser/grobid_client.py:**
    - Class `GrobidClient` with grobid_url from settings
    - `async def process_pdf(pdf_bytes: bytes) -> str`: POST to /api/processFulltextDocument with multipart form (input file + consolidateHeader=1 + consolidateCitations=1). Use httpx.AsyncClient with timeout=120s (PDF parsing can be slow). Return raw TEI XML string.
    - `async def is_alive() -> bool`: GET /api/isalive, return True if 200.
    - Wrap with tenacity retry: 2 attempts, 5s wait between retries, retry on httpx.TimeoutException

    **Create backend/app/services/pdf_parser/section_mapper.py:**
    - `def parse_tei_to_sections(tei_xml: str) -> dict`: Parse TEI XML using xml.etree.ElementTree with TEI namespace (http://www.tei-c.org/ns/1.0). Extract:
      - `title`: from //tei:titleStmt/tei:title
      - `abstract`: from //tei:abstract, join all paragraph text
      - `sections`: list of {heading: str, text: str} from //tei:body/tei:div, mapping tei:head to heading and concatenated tei:p to text
      - `references`: list of {title: str, authors: list[str], year: str | None, doi: str | None} from //tei:listBibl/tei:biblStruct
    - `def classify_section(heading: str) -> str`: Map section headings to standard categories: "introduction", "methodology", "experiments", "results", "discussion", "conclusion", "related_work", "other". Use keyword matching (e.g., "method" or "approach" -> "methodology", "experiment" or "evaluation" -> "experiments", "result" -> "results"). Handle both English and Chinese section headings (e.g., "方法" -> "methodology", "实验" -> "experiments").
    - `def build_structured_content(parsed: dict) -> dict`: Return final structure: {title, abstract, methodology (combined methodology sections), experiments (combined experiment sections), results (combined result sections), references, full_sections (all sections with classifications)}

  <action>
    **Create backend/app/services/pdf_parser/parser_service.py:**
    - Class `ParserService` with GrobidClient, httpx.AsyncClient, and db session
    - `async def parse_paper(paper_id: str, pdf_url: str) -> dict`:
      1. Download PDF from pdf_url using httpx (follow redirects, timeout 60s)
      2. Send PDF bytes to GrobidClient.process_pdf()
      3. Parse TEI XML via parse_tei_to_sections + build_structured_content
      4. Store parsed_content in papers table (update Paper.parsed_content JSONB column)
      5. Optionally upload raw PDF to SeaweedFS and store pdf_storage_key (use httpx PUT to SeaweedFS S3 endpoint; if SeaweedFS unavailable, log warning and skip)
      6. Return the structured content dict
    - `async def parse_paper_from_bytes(paper_id: str, pdf_bytes: bytes) -> dict`: Same pipeline but skip download step (for when PDF is uploaded directly)
    - Handle errors: GROBID failure -> return {error: "parsing_failed", detail: str}; download failure -> return {error: "download_failed", detail: str}

    **Create backend/app/routers/papers.py:**
    - POST /papers/{paper_id}/parse: Trigger parsing for a paper that has a pdf_url in the database. Requires the paper to exist in the papers table. Returns ApiResponse with parsed sections summary (title + section count + reference count).
    - POST /papers/parse-upload: Accept multipart file upload (PDF), create/find paper record, parse, and return structured content. This is for ad-hoc PDF parsing without a search.
    - Both endpoints are async but the actual parsing is slow (30-120s). For now, run synchronously in the request. In Phase 5, this will be moved to a Temporal workflow. Add a TODO comment noting this.

    **Update backend/app/main.py:**
    - Include papers router: `application.include_router(papers_router, prefix="/papers", tags=["papers"])`

    - Test ParserService: PDF download failure returns error dict
    - Test parse endpoint: POST /papers/{id}/parse returns 200 with section summary
    - Test parse-upload endpoint: POST /papers/parse-upload with file returns structured content
