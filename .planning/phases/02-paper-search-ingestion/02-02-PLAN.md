    - Implement two-phase search strategy:
      1. **First search:** Query Meilisearch with filters/sort. If results found, return them immediately (ms-level response).
      2. **Cache miss / first-time query:** If Meilisearch returns 0 results for this query, fall back to aggregator (fan-out to 4 sources), then index the fresh results in Meilisearch for next time. Return the fresh results.
    - This "index-on-search" pattern means Meilisearch gradually fills up as users search, without needing a bulk import.
    - Add offset/page param for pagination: page (int, default 1), limit (int, default 25)
