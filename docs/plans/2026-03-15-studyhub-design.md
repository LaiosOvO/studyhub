# StudyHub 学术研究社区平台 - 设计文档

> 日期: 2026-03-15
> 状态: 设计阶段

---

## 一、产品定位

**一句话**: 输入研究方向 → 自动搜索所有论文 → 分析改进空间 → 生成实验方案 → 一键自动执行实验 → 匹配跨学科合作者

**目标用户**: 高校教授/研究员（优先），研究生/博士生（覆盖）

**第一批测试方向**:
- ECG 自动诊断算法
- 起搏类型诊断系统
- 跨模态医疗大模型
- AI-ECG 年龄预测与生活习性分析系统
- 超声医学图像分割
- ECG 图像数字化工具

**商业模式**: Freemium（基础功能免费，自动实验/深度分析收费）

### 技术选型说明

| 组件 | 选择 | 理由 |
|------|------|------|
| Web 前端 | Next.js + React | SSR 对 SEO 友好，论文地图页面需要被搜索引擎收录 |
| 桌面应用 | Tauri | 比 Electron 小 15 倍 (10MB vs 150MB)，内存占用低 60% |
| API | FastAPI | Python 生态，与 ML 库无缝集成，异步性能好 |
| 工作流 | Temporal | Deep Research 和自动实验都是长时间多步骤任务，天然支持断点恢复 |
| 关系型数据库 | PostgreSQL | 用户/实验/方案元数据存储 |
| 图数据库 | Neo4j | 论文引用关系是典型图数据，多跳查询性能远超关系型 |
| 全文搜索 | Meilisearch | 支持中文分词，毫秒级搜索，比 ES 轻量 |
| 对象存储 | MinIO | S3 兼容，本地部署，存 PDF/模型/日志 |
| 分析统计 | ClickHouse | 列式存储，平台运营分析（热门方向、实验成功率） |
| LLM | Claude/GPT + 本地模型 | LLM Gateway 统一管理，支持降级和成本控制 |

## 三、Deep Research 引擎

Deep Research 是整个平台的基础能力，论文地图、方案生成、社区匹配都依赖它。

### 3.1 架构

```
用户输入

├─ 论文标题/DOI: "10.1038/s41591-023-02502-x"
└─ 作者: "某教授"
│
▼
┌────────────────────────────────────┐
│  Deep Research Agent               │
│                                    │
│  阶段1: 多源并行检索                │
│  ├─ OpenAlex API (2.5亿论文, 免费)  │
│  ├─ Semantic Scholar API            │
│  ├─ 知网爬虫                        │
│  ├─ 万方爬虫                        │
│  └─ arXiv API                      │
│                                    │
│  阶段2: 递归引用展开 (深度可配)      │
│  ├─ 被引论文 (谁引用了我)            │
│  ├─ 参考文献 (我引用了谁)            │
│  └─ 相关论文 (语义相似)             │
│                                    │

### 3.2 数据获取策略

| 数据源 | 方式 | 限额 | 覆盖范围 |
|--------|------|------|---------|
| OpenAlex | REST API (免费) | 无限 (polite pool 10/s) | 2.5亿论文，最全 |
| Semantic Scholar | REST API (免费) | 100 req/s (需 API key) | 2亿论文，引用关系好 |
| arXiv | OAI-PMH + API | 无限 | CS/物理/数学/医学预印本 |
| PubMed | E-utilities API | 10/s (有 key 后无限) | 生物医学，ECG 方向核心源 |
| 知网 | 爬虫 (CnkiSpider) | 注意反爬 | 中文论文 |
| 万方 | 爬虫 | 注意反爬 | 中文论文补充 |
| Crossref | REST API | polite pool | DOI 元数据 |
| Unpaywall | REST API | 100k/天 | 开放获取 PDF 链接 |

**医学方向特别说明**: PubMed 是 ECG/超声方向的核心数据源，OpenAlex 和 S2 作为补充。知网覆盖国内医学期刊论文。

### 3.4 API

```
POST   /api/deep-research/start
       body: {
         query: "ECG自动诊断算法",
         type: "direction" | "paper" | "author",
         depth: 2,
         sources: ["openalex", "pubmed", "s2", "cnki"],
         time_range: { start: 2018, end: 2026 },
         languages: ["zh", "en"]
       }
       返回: { task_id, estimated_time }

GET    /api/deep-research/{task_id}/status
       返回: {
         phase: "searching" | "expanding" | "analyzing" | "done",
         papers_found: 3847,
         papers_analyzed: 1200,
         progress_pct: 45,
         eta_seconds: 300
       }

WS     /api/deep-research/{task_id}/stream
       实时推送: 新发现论文、关系更新、阶段变更、AI洞察

GET    /api/deep-research/{task_id}/results
       返回: {
         papers: [...],
         graph: { nodes: [...], edges: [...] },
         insights: {
           trends: [...],
           gaps: [...],
           opportunities: [...]
         },
         rankings: [...]
       }

POST   /api/deep-research/{task_id}/expand
       body: { paper_id, depth: 1 }
       手动展开某个节点的引用

POST   /api/deep-research/{task_id}/refine
       body: { filters: { min_citations: 10 }, exclude: ["survey"] }
       缩小范围

GET    /api/deep-research/{task_id}/report
       返回: Markdown/PDF 格式的文献综述报告
```

### 5.3 测试场景: "ECG 自动诊断 — 起搏心律"

```

  - 起搏心律识别准确率目前最高 ~88% (2024)
  - 主要难点: 起搏钉波形态多样、与正常QRS混叠
  - 当前方法: 大多基于规则或浅层CNN
  - 空白: 没有人用 Transformer + 预训练做过起搏心律

生成方案:

┌──────────────────────────────────────────────────────┐
│  方案 A ★★★★★ 可行性高 · 预期提升 8-15%               │
│                                                      │
│  ECG Foundation Model 微调用于起搏心律分类              │
│                                                      │
│  假设: 在大规模ECG数据上预训练的Foundation Model,       │
│  微调后能显著提升起搏心律识别准确率                      │
│                                                      │
│  依据:                                                │
│  · ECG-FM (2024) 在心律失常分类上超越从头训练 12%       │
│  · 但尚未有人将其应用于起搏心律这一细分场景             │
│  · 起搏心律训练数据少, 预训练+微调范式天然适合小样本     │
│                                                      │
