# Plan 03.1-02 Summary: Google Scholar Enrichment, Paper Linking, Temporal Refresh

## What Was Built

Google Scholar enrichment service, scholar-paper linker with CJK-aware fuzzy matching, 4 new API endpoints, and Temporal ScholarRefreshWorkflow for batch operations.

## Key Decisions

- Inlined `_has_cjk` / `_is_cjk_char` in paper_linker.py to avoid import chain through patchright-dependent paper_search package
- `scholarly` library used via `asyncio.to_thread` (same pattern as pyalex in Phase 2)
- Module-level `asyncio.Semaphore(1)` + 5s cooldown for Google Scholar rate limiting
- CJK name matching uses exact match only (Chinese names too short for fuzzy); English uses rapidfuzz ratio >= 80
- Activities create their own DB sessions (Temporal isolation pattern)
