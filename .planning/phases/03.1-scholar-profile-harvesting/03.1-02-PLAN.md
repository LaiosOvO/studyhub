1. Create `backend/app/services/scholar/google_scholar_enricher.py`:
   - Class `GoogleScholarEnricher` that takes an `httpx.AsyncClient`
   - Method `async def search_scholar(self, name: str, institution: str | None = None) -> dict | None`:
     - Use the `scholarly` Python library (pip install scholarly) wrapped in `asyncio.to_thread` (same pattern as pyalex in 02-01: "pyalex (sync) with asyncio.to_thread for OpenAlex")
     - Search by name, optionally filter by affiliation containing institution
     - Return dict with: scholar_id, name, h_index, total_citations, interests, publications (list of recent 10 titles+years)
     - If not found, return None
     - Handle `scholarly` exceptions gracefully (MaxTriesExceededException, etc.)
   - Method `async def enrich_scholar(self, scholar: Scholar, db: AsyncSession) -> Scholar`:
     - Call search_scholar with scholar.name (try Chinese name first, then name_en if available)
     - If found, update scholar fields: h_index, total_citations, google_scholar_id
     - Append {"source": "google_scholar", "url": f"https://scholar.google.com/citations?user={scholar_id}"} to source_urls
     - Use SQLAlchemy update statement (immutable: build new values dict, execute update, refresh)
     - Return updated scholar (re-queried from DB after update)
   - Rate limiting: Use `asyncio.Semaphore(1)` at module level + 5s sleep between requests (Google Scholar is aggressive with rate limiting)
   - IMPORTANT: Use `scholarly.use_proxy(FreeProxy())` or configure a proxy if available. Without proxy, Google Scholar blocks after ~10 requests. Log a warning if no proxy configured.
   - Reference: Check if `scholarly` is suitable by reading its docs. If `scholarly` is too fragile, use direct httpx scraping of Google Scholar search results page as fallback (parse with lxml). Note this in code comments.
  </action>

   - Import and reuse CJK utilities from deduplicator: `from app.services.paper_search.deduplicator import _normalize_title, _fuzzy_match, _has_cjk`
   - Function `def normalize_author_name(name: str) -> str`:
     - Strip whitespace, normalize Unicode (NFKC)
     - For CJK names: remove spaces between characters (Chinese names have no spaces)
     - For English names: lowercase, collapse spaces
   - Function `def fuzzy_name_match(name_a: str, name_b: str, threshold: float = 80.0) -> bool`:
     - Normalize both names
     - For CJK: exact match after normalization (Chinese names are short, fuzzy matching would over-match). Also try matching with surname+given reversed.
     - For English: use rapidfuzz.fuzz.ratio > threshold
     - Handle mixed CJK/English: try both normalized forms
   - Function `async def link_scholar_papers(scholar: Scholar, db: AsyncSession) -> list[str]`:
     - Query all papers from DB (or limit to recent 1000 by year DESC)
     - For each paper, check if any author in paper.authors fuzzy-matches scholar.name or scholar.name_en
     - Return list of matched paper IDs
     - Does NOT create a join table yet -- returns paper IDs for the caller to use. The actual linking can be stored as a JSON field `linked_paper_ids` on Scholar, or in a separate `scholar_papers` table. Use JSON field on Scholar for now (simpler, sufficient for v1). Add `linked_paper_ids: Mapped[list] = mapped_column(JSON, default=list)` to Scholar model if not already present -- if model needs updating, do it via a simple ALTER or by adding the column in the migration from Plan 01.
