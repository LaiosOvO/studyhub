1. Create `backend/app/schemas/citation.py` with Pydantic models:
   - `CitationEdge(BaseModel)`: citing_paper: PaperResult, is_influential: bool = False, intents: list[str] = []
   - `CitationExpansionRequest(BaseModel)`: seed_paper_ids: list[str] (min 1, max 10), max_depth: int = 2 (1-3), budget_per_level: int = 50 (1-200), total_budget: int = 200 (1-1000)

2. Create `backend/app/services/citation_network/neo4j_client.py`:
   - `Neo4jClient` class wrapping `AsyncGraphDatabase.driver` (from `neo4j` package).
   - Constructor takes uri, user, password. Stores driver as `self._driver`.
   - `async close()` - closes driver.
   - `async verify_connectivity()` - calls `self._driver.verify_connectivity()`.
   - `async setup_schema()` - runs Cypher to create unique constraint on Paper.paper_id, indexes on Paper.doi and Paper.quality_score. Use `CREATE CONSTRAINT IF NOT EXISTS` and `CREATE INDEX IF NOT EXISTS`.
   - `async batch_merge_papers(papers: list[dict])` - UNWIND-based MERGE on Paper nodes. Each dict has: paper_id, title, doi, year, citation_count, s2_id, openalex_id. Use ON CREATE SET for all fields, ON MATCH SET for citation_count only.
   - `async batch_merge_edges(edges: list[dict])` - UNWIND-based MATCH+MERGE for CITES edges. Each dict has: citing_id, cited_id.
   - `async get_paper_neighborhood(paper_id: str, depth: int = 1) -> dict` - variable-length path query returning paper nodes and edges within N hops.
   - All queries run inside `async with self._driver.session() as session:` using `session.execute_write` or `session.execute_read` with transaction functions. The transaction function receives `tx` and must `await tx.run(query, **params)`, then collect results within the tx function (results consumed before tx closes).
   - IMPORTANT: Neo4j async driver tx functions must be async: `async def _tx_func(tx): result = await tx.run(...); return [record.data() async for record in result]`.

3. Add two new methods to `SemanticScholarClient` in `s2_client.py` (do NOT modify existing methods):
   - `async get_citations(self, paper_id: str, limit: int = 100, offset: int = 0) -> list[CitationEdge]`: GET `/paper/{paper_id}/citations` with fields `citingPaper.{S2_FIELDS},isInfluential,intents`. Parse each item's `citingPaper` via `_map_s2_to_paper`. Return list of CitationEdge. Add @retry decorator matching existing pattern.
   - `async get_references(self, paper_id: str, limit: int = 100, offset: int = 0) -> list[CitationEdge]`: GET `/paper/{paper_id}/references` with fields `citedPaper.{S2_FIELDS},isInfluential,intents`. Same pattern but uses `citedPaper` key. Add @retry decorator.
   - Both methods handle 404 gracefully (return empty list).

1. Create `backend/app/services/citation_network/expansion_engine.py`:
   - `async expand_citations(seed_paper_ids: list[str], s2_client: SemanticScholarClient, neo4j_client: Neo4jClient, max_depth: int = 2, budget_per_level: int = 50, total_budget: int = 200) -> CitationGraph`:
     - BFS loop over depth levels (0 to max_depth-1).
     - For each paper in the current queue, fetch citations AND references from S2 via the new get_citations/get_references methods.
     - Use `asyncio.Semaphore(1)` for S2 rate limiting (1 req/sec). Process papers sequentially within a level to respect rate limits.
     - Filter out already-visited papers (by s2_id).
     - Combine all new candidates, sort by citation_count descending (priority selection).
     - Take top min(budget_per_level, total_budget - total_added) papers.
     - Convert selected papers to dicts and call `neo4j_client.batch_merge_papers()` and `neo4j_client.batch_merge_edges()`.
     - Track edges as (citing_s2_id, cited_s2_id) tuples for each citation relationship discovered.
     - Update queue to be the s2_ids of selected papers for next level.
     - Break early if total_budget reached or queue empty.
     - Return CitationGraph with all discovered papers, all edges, and seed_ids.
   - Use existing deduplicator pattern: build new objects, never mutate inputs.
   - Log at each level: papers_in_queue, candidates_found, selected_count, total_so_far.
