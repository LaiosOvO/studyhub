# Plan 03.1-01 Summary: Scholar Model, Baike Scraper, Seed Import

## What Was Built

Scholar profile infrastructure: SQLAlchemy model, Pydantic schemas, Alembic migration, Baidu Baike scraper, seed JSON importer, and 6 REST API endpoints.

## Key Decisions

- Added `linked_paper_ids` JSON column to Scholar model upfront (needed by Plan 02 for paper linking)
- CSS selector fallback pattern reused from Phase 3 CNKI/Wanfang scrapers
- PostgreSQL `ON CONFLICT DO UPDATE` for idempotent seed imports and scrape upserts
- `ecg_ai_relevance` from seed data stored in `note` field (not a separate column)
